# Fixture Provenance and Policy

This directory contains BMP fixtures used by automated tests.

## Policy

- Only vendor fixtures with explicit, repo-safe licensing.
- If licensing is ambiguous, do not commit the binary file.
- Preserve provenance for every third-party fixture (source URL, version, license evidence, retrieval date).
- Prefer tiny deterministic fixtures for CI.
- Prefer generated fixtures for malformed/error-path cases where external licensing is uncertain.

## Source of Truth

The audit and source decisions are tracked in:

- `docs/test-fixtures-audit.md`
- `docs/test-fixture-sources.json`
- `docs/fixture-provenance.json`

## Third-Party Fixture Requirements

For every vendored third-party fixture, record:

- `origin`: canonical URL where file was obtained
- `source_version`: release/tag/date (if available)
- `license`: exact license classification used for this repository
- `evidence`: short quote + link supporting reuse
- `local_path`: fixture path in this repo

## Current Recommendation Snapshot

- Safe to vendor: curated files from The Bitmap Test Suite (SourceForge) and BMP Suite image outputs (excluding ICC-profile exceptions).
- Not safe to vendor by default: ConvertICO samples, Hlevkin collection, Burkardt mirror files without clarified file-level license, and upstream `bmp-js` fixtures without documented provenance.

## Suggested Layout

- `fixtures/third-party/bitmap-test-suite/` (public-domain files)
- `fixtures/third-party/bmpsuite/` (public-domain generated BMP files only; exclude ICC profile-related BMPs)
- `fixtures/generated/` (deterministic in-repo generated fixtures)
- `fixtures/generated-invalid/` (truncated/malformed fixtures generated by script)

## Notes for Contributors

- Do not add new binary fixtures without provenance updates in the audit docs.
- If you add generated fixtures, include the generator script and seed/parameters so files are reproducible.
